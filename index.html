Описать структуру НТML-документа, теги и их атрибуты. Семантическая разметка HTML-документа

HTML-документ состоит из тега <!DOCTYPE html>, корневого элемента <html>, который включает:

  • <head> — метаданные
  • <body> — контент

Теги — это элементы разметки (например, <p>, <h1>, <img>).

Атрибуты уточняют их свойства, например:

  • href в <a> указывает ссылку
  • src в <img> — путь к изображению

Описать формы HTML. Способы передачи данных на сервер, описать их

Создание форм

Формы создаются с помощью тега <form> с атрибутами:

  • action — адрес сервера, куда отправляются данные
  • method — способ отправки данных (GET или POST)

Основные элементы формы

  • <input> — поле ввода для:
    • текста (type="text")
    • паролей (type="password")
    • кнопок (type="button", type="submit")
    • чекбоксов (type="checkbox")
    • радиокнопок (type="radio")
  • <select> — выпадающий список

Каждый элемент формы имеет атрибут name для идентификации передаваемых данных.

Способы передачи данных

  • GET — данные передаются через строку URL:
    • Подходит для простых запросов (например, поиск)
    • Ограничен по объему
    • Данные видны в адресной строке
  • POST — данные отправляются в теле запроса:
    • Безопаснее, чем GET
    • Подходит для конфиденциальных данных
    • Используется для отправки больших объемов информации, включая файлы

Определить назначение каскадных таблиц стилей (CSS)

Каскадные таблицы стилей (CSS) предназначены для оформления и стилизации веб-страниц, созданных на HTML.

CSS управляет:

  • Цветами
  • Шрифтами
  • Размером и расположением элементов
  • Добавлением эффектов и анимаций

Использование CSS позволяет отделить дизайн от структуры документа, что упрощает его поддержку и изменение.

Что значит "каскадность"?

CSS использует систему приоритета и наследования, которая позволяет управлять стилями гибко и эффективно.

Описать синтаксис CSS

Синтаксис CSS состоит из селектора и блока объявлений в фигурных скобках {}.

Структура CSS-правила:

  • Селектор — определяет, к каким элементам применяются стили:
    • Теги (например, p)
    • Классы (через ., например, .classname)
    • ID (через #, например, #idname)
    • Комбинации селекторов (например, div p)
  • Блок объявлений — содержит стили, записанные в виде пар "свойство: значение", разделенных двоеточием : и заканчивающихся точкой с запятой ;.

Пример CSS-правила:

p { 
                color: blue; 
                font-size: 16px; 
            }

Этот код задает синий цвет и размер шрифта 16px для всех абзацев.

Способы добавления CSS:

  • Встроенный — в теге <style> внутри HTML.
  • Внешний — подключение через <link> к отдельному CSS-файлу.
  • Внутри атрибута style — непосредственно в теге элемента.
Описать способы подключения каскадных таблиц стилей, привести примеры

Способы подключения CSS

1. Внешний файл

CSS-код записывается в отдельный файл (например, styles.css) и подключается в <head> с помощью тега <link>:

<link rel="stylesheet" href="styles.css">

Преимущества: удобно для масштабируемых проектов, код хранится отдельно от HTML.

2. Встроенные стили в HTML

Стили размещаются внутри тега <style> в <head>:

<style>
            p { color: blue; }
            </style>

Когда использовать: подходит для небольших страниц или тестирования стилей.

3. Атрибут style

Стили задаются непосредственно в теге элемента через атрибут style:

<p style="color: red;">Текст</p>

Использование: для точечных изменений, но не рекомендуется для крупных проектов.

Дать определение селекторам. Привести примеры

Селекторы в CSS — это конструкции, которые определяют, к каким элементам HTML будут применяться стили.

Основные виды селекторов:

  • По тегу: применяется ко всем указанным тегам.
    • p { color: blue; }

    Все абзацы <p> будут синими.

  • По классу: применяется к элементам с указанным классом (через .).
    • .menu { font-size: 16px; }

    Все элементы с классом menu получат размер шрифта 16px.

  • По ID: применяется к элементу с указанным id (через #).
    • #footer { background: gray; }

    Элемент с id="footer" получит серый фон.

  • Универсальный селектор: применяется ко всем элементам.
    • * { padding: 0; }

    Убирает отступы у всех элементов.

  • Групповой селектор: позволяет применять стиль к нескольким элементам.
    • h1, h2 { color: green; }

    Все заголовки <h1> и <h2> будут зелеными.

  • Селектор потомков: применяется к элементам внутри другого элемента.
    • div p { font-weight: bold; }

    Все абзацы <p> внутри <div> станут жирными.

  • Дочерний селектор: применяется только к непосредственным дочерним элементам.
    • ul > li { list-style: none; }

    Сбрасывает маркеры у элементов <li>, которые являются прямыми потомками <ul>.

Дать определение псевдо-классам CSS. Привести примеры

Псевдо-классы в CSS — это специальные ключевые слова, добавляемые к селекторам через двоеточие :, которые определяют состояние или позицию элемента на странице.

Псевдо-классы позволяют применять стили в зависимости от взаимодействия пользователя или структуры документа.

Примеры псевдо-классов:

  • :hover — применяется при наведении на элемент.
    • button:hover { background: red; }

    Кнопка краснеет при наведении.

  • :active — применяется, когда элемент активен (например, при клике).
    • a:active { color: green; }

    Ссылка становится зеленой при клике.

  • :visited — применяется к посещенным ссылкам.
    • a:visited { color: purple; }

    Посещенные ссылки становятся фиолетовыми.

  • :first-child — применяется к первому элементу в родительском контейнере.
    • li:first-child { font-weight: bold; }

    Первый элемент списка становится жирным.

  • :last-child — применяется к последнему элементу в родительском контейнере.
    • li:last-child { color: blue; }

    Последний элемент списка становится синим.

  • :nth-child(n) — применяется к элементу, который является n-м ребенком родителя.
    • li:nth-child(2) { background: yellow; }

    Второй элемент списка становится желтым.

  • :focus — применяется к элементу, который находится в фокусе (например, поле ввода).
    • input:focus { border: 2px solid blue; }

    Поле ввода получает синюю рамку при фокусе.

Дать определение псевдо-элементам CSS. Привести примеры

Псевдо-элементы в CSS — это ключевые слова, добавляемые к селекторам через ::, которые позволяют стилизовать определенные части элемента или добавлять виртуальный контент, отсутствующий в HTML.

Псевдо-элементы применяются к конкретным фрагментам, таким как первая буква, строка или области перед и после содержимого.

Примеры псевдо-элементов:

  • ::before — добавляет контент перед содержимым элемента.
    • p::before { content: ">> "; color: red; }

    Добавляет красные символы ">>" перед текстом абзаца.

  • ::after — добавляет контент после содержимого элемента.
    • p::after { content: " [OK]"; color: green; }

    Добавляет зеленое " [OK]" после текста абзаца.

  • ::first-line — применяется к первой строке текста внутри элемента.
    • p::first-line { font-weight: bold; }

    Первая строка текста становится жирной.

  • ::first-letter — применяется к первой букве элемента.
    • p::first-letter { font-size: 2em; }

    Первая буква абзаца увеличивается в размере.

  • ::selection — применяется к выделенному тексту пользователем.
    • ::selection { background: blue; color: white; }

    Выделенный текст отображается с белым цветом на синем фоне.

Описать адаптивную верстку страниц при помощи CSS, назвать подходы к адаптивной верстке

Адаптивная верстка — это создание страниц, которые подстраиваются под размер экрана с помощью CSS.

Для этого используются:

  • Медиа-запросы (@media) для условий по ширине экрана.
    • @media (min-width: 600px) { /* стили для экранов шире 600px */ }
  • Относительные единицы (%, vw, rem) для гибкости размеров.
  • Flexbox и Grid для создания адаптивных макетов.

Подходы к адаптивной верстке:

  • Mobile First: базовые стили задаются для малых экранов, а затем через медиа-запросы добавляются изменения для больших экранов.
    • @media (min-width: 768px) { /* стили для экранов шире 768px */ }
  • Desktop First: базовые стили задаются для больших экранов, и изменения для малых экранов добавляются через медиа-запросы.
    • @media (max-width: 600px) { /* стили для экранов уже 600px */ }
Определить управление положением на странице (свойства left, top, z-index, position, visibility, overflow)

Управление положением элементов осуществляется с помощью свойств CSS, которые определяют расположение, видимость и поведение контента на странице.

Основные свойства для управления позиционированием:

  • position: задает тип позиционирования элемента.
    • position: static; /* по умолчанию */

    Возможные значения:

    • static — стандартное позиционирование;
    • relative — позиционирование относительно исходного положения;
    • absolute — абсолютное позиционирование относительно ближайшего родительского элемента с позиционированием;
    • fixed — фиксированное позиционирование относительно окна браузера;
    • sticky — элемент фиксируется на экране при прокрутке.

  • left, top: сдвигают элемент по горизонтали и вертикали относительно его позиции.
    • left: 20px; top: 10px;

    Работают с позиционированием relative, absolute, fixed, sticky.

  • z-index: управляет порядком наложения элементов.
    • z-index: 10;

    Чем выше значение, тем выше слой элемента.

  • visibility: управляет видимостью элемента.
    • visibility: hidden;

    Возможные значения:

    • visible — элемент видим;
    • hidden — элемент скрыт, но его место сохраняется на странице.

  • overflow: определяет поведение переполненного контента внутри элемента.
    • overflow: scroll;

    Возможные значения:

    • visible — контент выходит за пределы контейнера;
    • hidden — контент скрыт;
    • scroll — добавляются полосы прокрутки;
    • auto — полосы прокрутки появляются, если это необходимо.

Описать Flex модель

Flexbox — это модель CSS для гибкого размещения элементов в контейнере. Она задается с помощью свойства display: flex для родительского элемента.

Flexbox позволяет выстраивать дочерние элементы в ряд или столбец, автоматически подстраивая их под размер контейнера.

Основные свойства контейнера:

  • flex-direction — направление элементов внутри контейнера.
    • flex-direction: row; /* выстраивает элементы в строку */

    Возможные значения:

    • row — элементы выстраиваются по горизонтали;
    • column — элементы выстраиваются по вертикали.

  • flex-wrap — перенос элементов, если они не помещаются в контейнер.
    • flex-wrap: wrap; /* элементы переносятся на следующую строку */

    Возможные значения:

    • nowrap — элементы не переносятся;
    • wrap — элементы переносятся на следующую строку;
    • wrap-reverse — перенос элементов в обратном порядке.

  • justify-content — выравнивание элементов по главной оси.
    • justify-content: center; /* выравнивание по центру */

    Возможные значения:

    • flex-start — выравнивание по началу;
    • center — выравнивание по центру;
    • space-between — элементы равномерно распределены, первый элемент у начала, последний — в конце;
    • space-around — элементы с равными промежутками между ними;
    • space-evenly — равные промежутки между всеми элементами.

  • align-items — выравнивание элементов по поперечной оси.
    • align-items: center; /* выравнивание по центру поперечной оси */

    Возможные значения:

    • stretch — элементы растягиваются по высоте;
    • center — элементы выравниваются по центру;
    • flex-start — элементы выравниваются по верхнему краю;
    • flex-end — элементы выравниваются по нижнему краю;
    • baseline — элементы выравниваются по базовой линии текста.

Свойства элементов внутри контейнера:

  • flex-grow — определяет, насколько элемент может растягиваться, если есть свободное место.
    • flex-grow: 1; /* элемент растягивается, чтобы занять все доступное место */
  • flex-shrink — определяет, насколько элемент будет сжиматься при нехватке места.
    • flex-shrink: 1; /* элемент сжимается, если места не хватает */
  • flex-basis — задает начальный размер элемента перед распределением оставшегося пространства.
    • flex-basis: 200px; /* начальный размер элемента */
  • flex — это сокращение для комбинированных свойств flex-grow, flex-shrink и flex-basis.
    • flex: 1; /* элемент растягивается, занимая все доступное место */

Пример:

.container { display: flex; justify-content: center; }
.item { flex: 1; }
Описать Grid модель

Grid — это модель CSS для создания двумерных макетов, задаваемая с помощью свойства display: grid.

Она позволяет определять строки и столбцы, размещая элементы в ячейках сетки с точным контролем их положения.

Основные свойства контейнера:

  • grid-template-columns — задает ширину столбцов.
    • grid-template-columns: 100px 200px;

    Пример: задает первый столбец шириной 100px, второй — 200px.

  • grid-template-rows — задает высоту строк.
    • grid-template-rows: 50px 50px;

    Пример: задает две строки по 50px.

  • gap — расстояние между ячейками.
    • gap: 10px;

    Задает отступы между строками и столбцами.

  • grid-auto-rows — задает размер дополнительных строк, которые автоматически создаются.
    • grid-auto-rows: 100px;

    Пример: устанавливает высоту всех автоматически создаваемых строк в 100px.

  • repeat и minmax — упрощают настройку гибких сеток.
    • grid-template-columns: repeat(2, 1fr);

    Пример: repeat(2, 1fr) означает два одинаковых столбца, которые занимают одинаковое пространство.

Свойства элементов внутри контейнера:

  • grid-column и grid-row — указывают, сколько столбцов и строк занимает элемент.
    • grid-column: 1 / 3;
    grid-row: 1 / 2;

    Пример: элемент начинается с первого столбца и заканчивается на третьем.

  • span — задает протяженность элемента по строкам или столбцам.
    • grid-column: span 2;

    Пример: элемент занимает два столбца.

Пример:

.grid-container { display: grid; grid-template-columns: repeat(2, 1fr); gap: 10px; }
.item { grid-column: span 2; }
Описать создание анимации в CSS (переходы, ключевые кадры) и визуальные эффекты

Анимации в CSS создаются с помощью переходов (transition) и ключевых кадров (@keyframes). Они добавляют визуальные эффекты, улучшая оформление веб-страниц.

transition: плавное изменение свойств.

Пример:

transition: width 2s ease;

Ширина элемента меняется за 2 секунды с использованием функции ease для плавного эффекта.

@keyframes: задает этапы анимации.

Пример:

@keyframes move {
              0% { left: 0; }
              100% { left: 100px; }
            }

С помощью animation применяется анимация:

animation: move 3s;

Массивы в JavaScript

Массивы — это объекты, которые позволяют хранить несколько значений в одной переменной. Массивы могут содержать элементы разных типов, например, строки, числа и другие объекты.

Создание массива:

Массив можно создать следующим образом:

  • Используя литерал массива: let fruits = ["apple", "banana", "orange"];
  • Используя конструктор: let fruits = new Array("apple", "banana", "orange");

Доступ к элементам массива:

Для доступа к элементам массива используется индекс, начиная с 0:

fruits[0]; // "apple"

Изменение элементов массива:

Для изменения элемента массива достаточно присвоить новое значение по индексу:

fruits[1] = "grape";

Методы массивов:

  • push() — добавление элемента в конец массива.
  • unshift() — добавление элемента в начало массива.
  • pop() — удаление последнего элемента массива.
  • shift() — удаление первого элемента массива.
  • indexOf() — поиск индекса элемента в массиве.
  • forEach() — перебор элементов массива с выполнением функции для каждого элемента.
  • map() — создание нового массива с измененными элементами.

Пример работы с массивами:

let fruits = ["apple", "banana"];
            fruits.push("orange");     // добавление в конец
            fruits.unshift("kiwi");    // добавление в начало
            fruits.pop();              // удаление с конца
            fruits.shift();            // удаление с начала
            fruits.forEach(function(fruit) { 
                console.log(fruit); 
            });

События в JavaScript

События — это действия пользователя, такие как клики, ввод текста, отправка форм и другие, на которые JavaScript может реагировать и выполнять соответствующие действия.

Способы назначения обработчиков событий:

  • addEventListener — позволяет добавлять несколько обработчиков для одного события. Это более гибкий и современный способ.
  • Свойства события (например, onclick) — позволяет назначить только один обработчик для события. Используется реже, так как не поддерживает несколько обработчиков для одного события.
  • Встраивание в HTML — например, <button onclick="alert('Hello')">, но этот способ не рекомендуется, так как снижает читаемость и масштабируемость кода.

Объект event:

Объект event предоставляет информацию о событии и позволяет управлять его поведением:

  • preventDefault() — отменяет стандартное поведение события, например, предотвращает отправку формы.
  • stopPropagation() — останавливает всплытие события, предотвращая его передачу на другие элементы.

Примеры событий:

  • Клик: element.addEventListener('click', function() { alert('Клик!'); });
  • Ввод текста: input.addEventListener('input', function() { console.log('Текст введен!'); });
  • Отправка формы: form.addEventListener('submit', function(event) { event.preventDefault(); console.log('Форма отправлена!'); });

Объект event в JavaScript

Объект event предоставляет информацию о событии, которое произошло. Он включает различные свойства, которые помогают понять контекст события и управлять его поведением.

Основные свойства объекта event:

  • event.type — тип события (например, click, keydown).
  • event.target — элемент, на котором произошло событие.
  • event.preventDefault() — отменяет стандартное поведение события (например, отменяет отправку формы).
  • event.stopPropagation() — останавливает всплытие события, предотвращая его распространение на родительские элементы.

Пример обработки события:


            element.addEventListener("click", function(event) {
                console.log("Событие:", event.type);
            });

Основные типы событий:

  • Мышь: click, dblclick, mousemove
  • Клавиатура: keydown, keyup
  • Формы: submit, input, change
  • Загрузка: DOMContentLoaded, beforeunload

Определить promises и fetch в JavaScript

Асинхронные операции в JavaScript

Promise:

Promise — это объект для работы с асинхронными операциями. Он имеет три состояния:

  • Pending: Ожидание выполнения.
  • Fulfilled: Операция завершена успешно.
  • Rejected: Операция завершена с ошибкой.
Создание Promise:
                
                let promise = new Promise((resolve, reject) => {
                    resolve("Успех"); // Успешное выполнение
                    // reject("Ошибка"); // Ошибка выполнения
                });
Обработка Promise:
                
                promise
                    .then(result => console.log(result)) // Обрабатываем успешный результат
                    .catch(error => console.log(error))  // Обрабатываем ошибку
                    .finally(() => console.log("Завершение")); // Выполняется всегда

Fetch API:

Fetch API — это метод для отправки HTTP-запросов. Возвращает Promise.

Пример использования Fetch:
                
                fetch("https://jsonplaceholder.typicode.com/todos/1")
                    .then(response => response.json()) // Преобразуем ответ в JSON
                    .then(data => console.log(data))    // Обрабатываем данные
                    .catch(error => console.log(error)); // Обрабатываем ошибки

Async/Await:

Async/await — это синтаксический сахар для работы с асинхронными функциями, улучшает читаемость.

Пример использования Async/Await:
                
                async function getData() {
                    try {
                        let response = await fetch("url"); // Ждем ответа от сервера
                        let data = await response.json();  // Ждем преобразования в JSON
                        console.log(data);                  // Выводим данные
                    } catch (error) {
                        console.log(error.message);         // Обрабатываем ошибки
                    }
                }

Обработка ошибок в JavaScript

try...catch

Механизм для обработки ошибок в JavaScript. Блок try выполняет код, а блок catch ловит ошибки, если они возникли.

Пример использования try...catch:
                
                try {
                    // Код, который может вызвать ошибку
                    let x = y; // Ошибка, так как переменная y не определена
                } catch (e) {
                    // Обработка ошибки
                    console.log(e.message); // Выводим сообщение об ошибке
                }

finally

Блок finally всегда выполняется, независимо от того, была ли ошибка или нет.

Пример с finally:
                
                try {
                    // Код, который может вызвать ошибку
                    let x = y; // Ошибка, так как переменная y не определена
                } finally {
                    // Код, который выполняется всегда
                    console.log("Всегда выполняется"); // Это сообщение будет выведено, даже если была ошибка
                }

Создание и выброс ошибки с throw

Для создания и выброса ошибок используется throw с объектом new Error().

Пример с throw:
                
                function проверитьВозраст(возраст) {
                    if (возраст < 18) {
                        throw new Error("Вам нет 18 лет!"); // Генерация ошибки, если возраст меньше 18
                    }
                    return "Доступ разрешён";
                }
            
                // Пример вызова
                try {
                    console.log(проверитьВозраст(16)); // Возраст 16 лет
                } catch (e) {
                    console.log(e.message); // Вывод сообщения ошибки: "Вам нет 18 лет!"
                }

Обработка ошибок в async/await

Ошибки в асинхронных функциях обрабатываются с использованием try...catch внутри async функции.

Пример с async/await:
                
                async function f() {
                    try {
                        let response = await fetch("url"); // Ожидание ответа от сервера
                        let data = await response.json();  // Преобразование ответа в JSON
                    } catch (e) {
                        console.log(e); // Обработка ошибок, если fetch или .json() вызвали исключение
                    }
                }

JSON (JavaScript Object Notation)

JSON — это текстовый формат для хранения и передачи данных, который является независимым от языка программирования. Он представляет собой строку, которую легко преобразовать в объект JavaScript.

Пример структуры JSON:
                
                {
                    "имя": "Алиса",
                    "возраст": 25
                }
Преобразование JSON в объект и наоборот:
                
                // Преобразование JSON строки в объект
                let jsonString = '{"имя": "Алиса", "возраст": 25}';
                let obj = JSON.parse(jsonString);
                console.log(obj);  // Выводит: { имя: "Алиса", возраст: 25 }
            
                // Преобразование объекта в JSON строку
                let jsonStringified = JSON.stringify(obj);
                console.log(jsonStringified);  // Выводит: '{"имя":"Алиса","возраст":25}'

Работа с Cookie в PHP

Создание Cookie

Устанавливаем cookie с временем жизни 1 час:


                setcookie("user", "Alex", time() + 3600); // Хранится 1 час

Чтение Cookie

Получаем значение Cookie в PHP:


                if (isset($_COOKIE["user"])) {
                    echo "Привет, " . $_COOKIE["user"];
                } else {
                    echo "Куки не установлены.";
                }

Удаление Cookie

Удаляем cookie, установив время истечения в прошлом:


                setcookie("user", "", time() - 3600);

Безопасные Cookie

Настройки для защиты от XSS и передачи только по HTTPS:


                setcookie("secureUser", "Alex", time() + 3600, "/", "", true, true);
  • Параметр 4 ("/") — доступен на всем сайте.
  • Параметр 5 ("") — указывает домен (по умолчанию текущий).
  • Параметр 6 (true) — требует HTTPS.
  • Параметр 7 (true) — делает cookie доступным только через HTTP (защита от XSS).

            class Transport {
                #brand;
                #capacity;
                #power;
                #speed;
                
                constructor() {
                    this.#brand = "бам бам бим бим";
                    this.#capacity = 0;
                    this.#power = 0;
                    this.#speed = 0;
                }
                
                setProperties(brand, capacity, power, speed) {
                    this.#brand = brand;
                    this.#capacity = capacity;
                    this.#power = power;
                    this.#speed = speed;
                }
                
                displayProperties() {
                    return `Марка: ${this.#brand}<br>Вместительность: ${this.#capacity} пассажиров<br>Мощность: ${this.#power} л.с.<br>Скорость: ${this.#speed} км/ч`;
                }
            }
            
            class Airplane extends Transport {
                #engines;
                #flightAltitude;
                
                constructor(brand = "Unknown", capacity = 0, power = 0, speed = 0, engines = 1, flightAltitude = 0) {
                    super();
                    this.setProperties(brand, capacity, power, speed);
                    this.#engines = engines;
                    this.#flightAltitude = flightAltitude;
                }
                
                displayProperties() {
                    return `${super.displayProperties()}<br>Количество двигателей: ${this.#engines}<br>Высота полёта: ${this.#flightAltitude} м`;
                }
            }
            
            function savePlane() {
                // Берём значения из формы
                const brand = document.getElementById("brand").value;
                const capacity = parseInt(document.getElementById("capacity").value);
                const power = parseInt(document.getElementById("power").value);
                const speed = parseInt(document.getElementById("speed").value);
                const engines = parseInt(document.getElementById("engines").value);
                const flightAltitude = parseInt(document.getElementById("flightAltitude").value);
            
                const plane = new Airplane(brand, capacity, power, speed, engines, flightAltitude);
                document.getElementById("output").innerHTML = plane.displayProperties();
            }
